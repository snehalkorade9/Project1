PEP8

PEP stands for Python Enterprise Proposal. PEP 8 is a document that provides various guidelines to write the readable in Python. PEP 8 describes how the developer can write code which can be understood globally. PEP aims to enhance the readability and consistency of code.

Pylint
    Pylint is a tool that follows the style and rules recomanded by PEP8. Pylint checking your code for basic stylistic mistakes. It doesn't verify that your code works, it simply checks that it looks as good as possible and is readable by others.

Testing Types Testing can be done in 2 ways manually and automation. The best way is through automation, as this reduces manual repetation of work. Automation testing reduces time and efforts of testing without comprmising on the quality. To achieve different goals there are multiple types of automation testing like Unit, Integration, Smoke, Regression, API, Acceptance, Performance, Security,

AAA To write good unit testing we need to follow some basic pattern called AAA.
  • Arrange- Arrange pre-requsites required for testing
  • Act- Perform the main thing to be tested
  • Assert- Verify if the act matches the actual behaviour
