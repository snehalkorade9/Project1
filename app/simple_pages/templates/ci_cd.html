Continuous Integration and Deployment

A CI/CD pipeline is a series of steps that must be performed in order to deliver a new version of software. Continuous integration/continuous delivery (CI/CD) pipelines are a practice focused on improving software delivery using either a DevOps or site reliability engineering (SRE) approach.
Git Actions

Workflow

The Workflow are set using git actions. Git actions helps in defining the proccess. What and when to execute is defined here
Project structure
    Workflow
  • dev.yml- This file executed on when pull request is hit. It first runs the pytest to check for any test faliur and deployes the code to deployment environment
  • Dev.yml- This file runs the pytest and deployes the code to development environment when Push request is hit
  • Prod.yml- This file runs the pytest and deployes the code to production environment when pull request is successful and Merge is hit. This file builds container and then uploads the image to docker hub

      • CI/CD Steps
  • Clone a Project
  • Create a branch
  • Make required changes to the branch
  • Open Git Hub, create Repo and copy the path
  • Update the copied path in Git Remote
  • Login to Dockerhub and create Repository
  • Mention the Docker repo name in prod.yml
  • Login to Heroku create dev and prod apps and link it to Git Repo, create authorization key.
  • In Git Sectet setting mention the key generated above
  • Pipeline is now ready
  • Commit and push the changes
  • The changes once successfully, docker image is generated and deployed to both Dev and Production servers
