Docker

Docker is an open platform for developing, shipping, and running applications. Docker enables you to separate your applications from your infrastructure so you can deliver software quickly.

Docker objects

Image

A Docker image is a read-only template that contains a set of instructions for creating a container that can run on the Docker platform

Container

A Docker container image is a lightweight, standalone, executable package of software that includes everything needed to run an application: code, runtime, system tools, system libraries and settings.

Getting Started

docker run

Runs a command in a new container.
$ docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

docker start

Starts one or more stopped containers
$ docker start [OPTIONS] CONTAINER [CONTAINER...]

docker stop

Stops one or more running containers
$ docker stop [OPTIONS] CONTAINER [CONTAINER...]

docker build

Builds an image form a Docker file
$ docker build [OPTIONS] PATH | URL | -

docker pull

Pulls an image or a repository from a registry<
$ docker pull [OPTIONS] NAME[:TAG|@DIGEST]

docker push

Pushes an image or a repository to a registry
$ docker push [OPTIONS] NAME[:TAG]

docker ps

This command is used to list the running containers
$ docker ps

docker compose

This command is used to run multiple containers as a single service
$ docker compose up --build

Docker Getting started

1. Before starting, Install git bash, pycharm, docker desktop
2. create Git, Jetbrains, Docker accounts and link them
3. On Git Bask or command Prompt Navigate to the App folder of your project file
4. To run the basic test run, this will create a container
docker run -dp 80:80 docker/getting-started
You can view the container using command- docker ps. Kill the container using docker kill container ID
6. To start up with Clone a repository - https://github.com/docker/getting-started/tree/master/app
7. Create DockerFile(without any extention) in location same as package.jason
8. Start application container using command
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
9. Now go to https://localhost/ to view the application
